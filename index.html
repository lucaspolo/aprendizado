O que é esta lista?

Esta simples lista serve apenas para guardar o que estou aprendendo com a minha experiência diária. O aprendizado de cada desafio novo costuma virar um novo tópico aqui.

Buscando soluções

  • Reduzir dimensões pode ajudar, generelizando o problema.

Na hora do desenvolvimento, não se esqueça:

  • Documente sua API (docstrings, swagger, manual e etc);
  • Remova código comentado antes do commit;
  • Não retorne null, não passe null.
  • Não use Magic Numbers;
  • Crie testes unitários:
    • Teste todas as condições (branchs);
    • Arrange -> Act -> Assert
    • Valide todos os valores, não apenas as quantidades
  • Faça testes de integração, veja o código rodar como seu cliente verá;
  • Programação funcional é uma boa mesmo para não linguagens não funcionais, fique de olho neste conceito;

Troubleshooting (caçando bugs):

  • Problemas devem ser encarados com frieza, não entre em pânico!
  • Confira valores por inteiro, não compare por partes:
    • Ex: 1923293 é diferente de 1926293
  • Confira as variáveis de configuração (ambiente, arquivos de configuração e etc)
